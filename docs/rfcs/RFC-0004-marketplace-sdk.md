# RFC-0004: Storage Marketplace SDK

**Status:** Proposed (Sprint C)
**Author:** MeshWeb Core Team
**Created:** 2026-07-20

## Summary
Proposes the introduction of the Application Layer Marketplace through an external SDK, explicitly isolating economic functions (pricing, escrow, reputation) from the Core Storage Protocol (V1).

## Motivation
MeshWeb must support financial incentives for nodes hosting data. However, injecting economic rules directly into the storage layer violates the *Payment Verification Invariant* and the *Marketplace Excluded* constitution principles. We need a secure abstraction layer.

## Specification
- **SDK Abstraction:** Marketplace features will be shipped as an SDK that wraps the core protocol daemon.
- **Deal Negotiation:** Out-of-band communication for establishing `PricePerGBDay`.
- **Payment Verification:** Proof-of-Storage generated by the Protocol V1 engine will be consumed by the Marketplace SDK to authorize settlement vouchers.
- **Fiat-Denominated Economics:** Pricing logic must remain compatible with fiat-pegged stable values, deferring complex tokenomics.

## Drawbacks
Requires application developers to manage two separate conceptual domains (Storage State vs. Economic State), potentially increasing the complexity of building a fully-featured storage client.
